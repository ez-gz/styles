Design System v1.0 — 2026

Typewriter

A minimal, monospace-first design system built on hard edges, tight letter-spacing, and motion through text — not decoration.

01 — Philosophy

Typewriter is a brutalist web design system built around the monospace font as a singular aesthetic choice. Everything else follows from that — sharp corners because serif type has serifs, tight spacing because type has metrics, motion through text because the typewriter advances one character at a time.

  • One font family, used completely Courier Prime or IBM Plex Mono at all sizes, weights, and cases. No display font, no sans fallback, no exceptions.
  • No rounded corners, ever border-radius: 0 on all interactive elements. Softness is never the goal. Hard edges reinforce the grid.
  • Text is the interface Loading states are words ("waiting...", "loading..."). Errors are sentences. Labels are uppercase type, not icons. The UI communicates through writing, not symbols.
  • Motion through opacity and text, not transforms Content reveals via opacity transitions. Waiting states animate the text itself (dot cycling). No slides, bounces, or fades from below.
  • Two colors: ink and paper #111 on #fff. A third color (#7a0000) exists only for errors. Everything else is a tint of black.
  • Typography is layout Letter-spacing, text-transform, and size differences create the visual hierarchy. There are no background colors for grouping — borders do that.
  • Density without clutter Spacing is generous enough to breathe but never decorative. Every pixel of margin serves legibility.

06 — Animation

This system uses the minimum motion necessary to communicate state. No entrance animations, no scroll-triggered effects, no micro-interactions. Animation is reserved for: (1) loading states that tell the user something is happening, (2) content reveal that signals new information has arrived.

  • Dot cycling — tw-dots (CSS) or JS interval at 450ms Appends ".", "..", "..." in a loop. Used for all waiting/loading states. Never use a spinner graphic — text communicates the same information.
  • Cursor blink — tw-cursor Block cursor (▎) that blinks via step-end animation. Use at the end of text being typed out. Remove once the reveal is complete.
  • Content reveal — tw-story-revealing → tw-story-complete Opacity transition at 140ms linear. Content fades from 0.92 to 1 as it arrives, signaling "this just changed." Never animate from 0 — the text should already be present.
  • Disabled state — opacity 0.6 No transition. The control is either on or off.
  • Hover state — background: #f5f5f5 No transition. Instant, like pressing a key.
07 — Porting Guide

To apply Typewriter to any new project, follow these steps in order. The goal is to completely replace the existing visual language without touching logic.

1
Load the font Add Courier Prime and IBM Plex Mono via Google Fonts. These are the only fonts the system uses.
<link href="https://fonts.googleapis.com/css2?family=Courier+Prime:wght@400;700&family=IBM+Plex+Mono:wght@400;500;600&display=swap" rel="stylesheet" />
2
Copy the CSS block Paste everything between the START / END markers in this file's <style> block into your global stylesheet. This defines all tokens and component classes. Do not modify it — override with additional rules if needed.
3
Remove all border-radius Search for border-radius in existing CSS and set everything to 0. If using Tailwind, add borderRadius: { DEFAULT: '0', none: '0' } to the theme and replace all rounded-* classes with rounded-none.
4
Replace color tokens Map your existing color system to Typewriter tokens:
primary / brand → --tw-ink (#111) background / surface → --tw-paper (#fff) secondary / subtle → --tw-muted (#555) body text → --tw-dim (#333) border / divider → --tw-rule (#d0d0d0) error / danger → --tw-error (#7a0000) hover / selected → --tw-hover / --tw-active-bg
5
Replace loading UI Remove spinners, skeleton screens, and progress bars. Replace with .tw-waiting text + .tw-dots CSS animation, or a JS interval that cycles "." / ".." / "..." every 450ms.
6
Replace icon labels with text labels Any icon-only button should become a text button. Any icon + label should drop the icon. Text-transform: uppercase + letter-spacing does the visual work icons used to do.
7
Apply tw-eyebrow to all section headers Any h2, h3, or label that acts as a section header should become a .tw-eyebrow or .tw-label. This replaces decorative dividers with typographic hierarchy.
8
Convert cards to tw-card / tw-panel Remove background colors, box shadows, and rounded corners from cards. A card is just a 1px solid border with 10–12px padding. Active/selected state is border-color: var(--tw-ink); background: var(--tw-active-bg).
9
Tailwind projects: extend theme
// tailwind.config.js theme: { extend: { fontFamily: { mono: ['"Courier Prime"', '"IBM Plex Mono"', 'ui-monospace', 'monospace'], }, borderRadius: { DEFAULT: '0', none: '0', sm: '0', md: '0', lg: '0', xl: '0', full: '0', }, colors: { ink: '#111', paper: '#fff', muted: '#555', dim: '#333', rule: '#d0d0d0', hover: '#f5f5f5', activeBg: '#f7f7f7', error: '#7a0000', }, }, }
10
Final check — the "typewriter test" Open the app and ask: does it look like something you could print on paper and read as a document? If yes, it passes. If there are gradients, shadows, icons, or rounded UI chrome, they need to go.
